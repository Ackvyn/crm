Deploy your own Worker

Product overview: Worker. The Worker is yours. It does not serve the CRM GUI — only API routes (/v1/…, WebSockets, email, hub). The admin console and embeds load from the Ackvyn CRM CDN; agents open the console from your operator site after auth.

Uses Cloudflare’s Deploy to Cloudflare flow. The button points at Ackvyn/crm → worker/ — Cloudflare clones that folder only, reads its wrangler.toml, provisions the Durable Object, and sets up Workers Builds. The monorepo root is not the Worker project (no public root wrangler.toml). You add secrets and edit vars after deploy. For Free-plan request budgets, see Free tier & usage.

If you connect Ackvyn/crm manually in the dashboard (Workers Builds → Connect), set Root directory to worker. Connecting the repo root will not find a committed Wrangler config.

What you configure

  • wrangler.toml — Worker name, Durable Object binding, cron (reviews sync). Template ships with example vars only.
  • Vars — ALLOWED_SITE_ORIGINS (your operator site + clients), CRM_GIT_REPO / CRM_GIT_PATH_PREFIX, CRM_INBOUND_EMAIL_SITE, and AUTH_WORKER_URL (your auth Worker for console sessions)
  • Secrets — left blank in the template (Deploy must not look like it includes a shared token). Set after deploy:
    • GITHUB_TOKEN — your GitHub PAT (Contents on CRM_GIT_REPO)
    • CRM_DATA_PASSPHRASE — long random string for encrypting crm-data/ (same value in the CRM console). Not a short password — generate one here or run openssl rand -base64 32

Generate a data passphrase

Browser-only: 32 random bytes as base64. Paste into wrangler secret put CRM_DATA_PASSPHRASE and the console. Keep a backup — you cannot decrypt old files without it.

After deploy

  1. Note your Worker URL: https://<name>.<subdomain>.workers.dev
  2. Open the CRM console from your authenticated operator site (iframe or link after auth) and set that URL as the API base for the session / site settings.
  3. Create a site key in the hub; add client site origins.
  4. Give clients embed snippets with data-crm="https://YOUR-WORKER…".
  5. Optional: wire Email Routing → this Worker.

Local / CLI alternative

git clone https://github.com/Ackvyn/crm.git
cd crm/worker
npm install
npx wrangler deploy
npx wrangler secret put GITHUB_TOKEN
npx wrangler secret put CRM_DATA_PASSPHRASE
