Getting started

You are the operator: you deploy a Worker, gate the console with auth, and open it from your operator site. Client websites only load scripts.

  1. Deploy your Worker (see Worker setup for the Deploy to Cloudflare button or wrangler).
  2. Set up Auth & GitHub OAuth so agents can sign in.
  3. Open the console from your authenticated operator site (iframe or link) and point it at your Worker URL — not as a public CDN bookmark for strangers.
  4. Create a site key in the hub; allow the client’s origin.
  5. Give them an embed with src = this CDN and data-crm = your Worker.
  6. Optional: Email Routing, Google reviews, and Google Analytics in Settings.

Designed for Cloudflare Workers Free budgets — see Free tier & Worker usage.

Custom domain for this CDN

Attach a domain to the GitHub Pages site (e.g. crm.ackvyn.org). Script src hosts change; each operator’s data-crm still points at their own Worker.
