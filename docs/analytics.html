Google Analytics

Ackvyn CRM’s Analytics panel reads your GA4 property through the Google Analytics Data API. The Worker stores a service-account key (encrypted) and proxies reports for signed-in agents. CRM does not replace the GA tag on the website — collection still happens in GA4 as usual.

What you need

  • A GA4 property already receiving hits from the client site (gtag / GTM / equivalent).
  • The numeric Property ID (Admin → Property settings) — not the G-… Measurement ID.
  • A Google Cloud service account JSON key with access to that property (Viewer is enough).
  • Worker secret CRM_DATA_PASSPHRASE set (same secret used for encrypted CRM data) so the key can be stored at rest.

1. Collect traffic in GA4

Install GA4 on the public site first. Until the property has data, the CRM Analytics panel will be empty even if the connection succeeds.

2. Enable the Analytics Data API

  1. Open Google Cloud Console.
  2. Select (or create) a project.
  3. Enable Google Analytics Data API (API Library → search “Analytics Data API”).

3. Create a service account

  1. IAM & Admin → Service Accounts → Create.
  2. Create a JSON key and download it (keep it private).
  3. Note the client_email in that file (looks like …@….iam.gserviceaccount.com).

4. Grant the service account access in GA4

  1. Open GA4 → Admin → Property access management.
  2. Add the service account email with role Viewer (or higher).

Without this step the Worker authenticates to Google but reports fail with a permission error.

5. Find the Property ID

GA4 Admin → Property settings → Property ID (digits only, e.g. 123456789). Do not paste the Measurement ID (G-XXXX).

6. Connect in the CRM console

  1. Sign in to your operator console (see Auth).
  2. Open Settings → Analytics.
  3. Paste the Property ID and the full service-account JSON.
  4. Save. Status should show Connected plus the client email.

The JSON key is stored encrypted in your site’s Durable Object. It is never shown back in the browser after save — leave the JSON field blank when you only need to change the Property ID.

7. View reports

Open the console Analytics section. Choose 7d / 28d / 90d. Typical totals: sessions, users, page views, plus countries and top pages from the Data API.

API (your Worker)

Session-gated (agent Bearer token required):

GET  /v1/{site}/analytics/ga/config
PUT  /v1/{site}/analytics/ga/config
DELETE /v1/{site}/analytics/ga/config
GET  /v1/{site}/analytics/ga/report?range=7d|28d|90d

Troubleshooting

  • Invalid property id — you used G-… instead of the numeric Property ID.
  • Permission / 403 from Google — service account email missing from GA4 Property access, or Analytics Data API not enabled on the Cloud project that owns the key.
  • Empty charts — property has no data yet, wrong property, or date range with no traffic.
  • Save fails — confirm CRM_DATA_PASSPHRASE is set on the Worker and you are signed in with a valid CRM session.

Live chat “On site” presence is separate: that comes from the CRM embed + Worker Durable Object, not from Google Analytics.
