Documents & e-sign

The console Documents section is an encrypted workspace library in git — separate from ticket-only attachments. Use it for templates, field layouts, and retained signed copies.

Library & folders

  • Upload PDFs (and other files) into the site’s Documents store.
  • Create folders / subfolders; drag files between folders.
  • Move, copy, duplicate, rename, and delete via the file menu.

Stored under your data repo as:

crm-data/{site}/documents/
  manifest.json.enc
  {document-id}.enc

PDF fields & share links

  • Place fields — layout signature / text / date fields on a PDF (source stays editable).
  • Create share link — Worker e-sign session for a contact + ticket; signer opens the link, completes required fields.
  • When signing finishes, the signed PDF attaches to the ticket (with audit note / hashes) — not automatically into Documents.

Everyday paperwork with an audit trail — not a government-certified / qualified electronic signature. Use a DocuSign-class vendor when a counterparty requires it, then attach the finished PDF to the ticket.

Tickets ↔ Documents

  • Ticket files stay on the ticket until you choose ⋮ → Save to Documents (name + folder). Signed copies can retain contact + e-sign audit on the library record.
  • Contact detail shows a Signed documents list (deduped across ticket + library). Open ticket and/or Open in Documents when each still exists.
  • Deleting a ticket that still has a signed PDF not saved to Documents prompts an extra warning first.

Worker vs browser

Library CRUD is browser → GitHub (like contacts/tickets). Creating and completing e-sign sessions goes through your CRM Worker (/sign/sessions).
