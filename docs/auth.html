Auth & GitHub OAuth

The CRM console is not a public app on this marketing site. Agents open it from a private page on their site (iframe or link) after signing in with GitHub — the same pattern many people already use for a CDN-served git CMS such as Sveltia CMS.

CRM Worker vs auth Worker

The CRM Worker you deploy from this project is API only. It does not implement GitHub OAuth itself. It verifies sessions by calling an auth Worker you configure with AUTH_WORKER_URL.

  1. Auth Worker — hosts the GitHub OAuth app, allowlisted logins, short-lived sessions (and, when used with CRM, optional GitHub token + data passphrase for encrypted crm-data/). If you already run an auth Worker for a git-based CMS, you can usually reuse that same Worker.
  2. CRM Worker — chat, tickets, email, git sync. Set AUTH_WORKER_URL to your auth Worker so session checks stay consistent.

GitHub OAuth app

  1. Create a GitHub OAuth App (Developer settings). Homepage URL = your site; callback URL = your auth Worker’s OAuth callback path.
  2. Store client id/secret on the auth Worker (secrets). Allowlist agent GitHub usernames.
  3. CRM console sign-in uses one authorize that can return a session plus a scoped token for git writes to crm-data/.

Where the console lives

  • GUI script: this CDN (https://crm.ackvyn.org/cdn/ackvyn-crm-console.js) — same idea as Sveltia’s sveltia-cms.js. Not linked in the public top nav.
  • Operator UX: host a thin HTML page on your site (e.g. /crm/index.html) that loads the script. Pass your Workers as attributes: 
    <script
  src="https://crm.ackvyn.org/cdn/ackvyn-crm-console.js"
  data-api="https://YOUR-CRM.workers.dev"
  data-auth="https://YOUR-AUTH.workers.dev"
  data-site="yoursite"
></script>
    The browser origin is your site, so CORS / OAuth allowlists only need your domain — not this CDN.
  • Public chat widgets still use embed.js + data-crm — no OAuth required for visitors.

CRM Worker config

# wrangler.toml [vars]
AUTH_WORKER_URL = "https://YOUR-AUTH.workers.dev"

# Secrets on the CRM Worker
# npx wrangler secret put GITHUB_TOKEN
# npx wrangler secret put CRM_DATA_PASSPHRASE
