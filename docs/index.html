Setup and reference for operators. Product story lives on What is, Why, and Worker — this section is the how-to wiki.

Guides

  • Getting started — operator checklist
  • Worker setup — Deploy to Cloudflare, secrets, vars
  • Free tier & usage — client-side first, what hits the Worker, ~100k req/day
  • Auth & OAuth — GitHub OAuth; open console from your operator site
  • Email routing — Cloudflare Email → Worker
  • Embeds — chat + reviews script tags
  • Reviews — public JSON + loader
  • Analytics — connect GA4 for the console Analytics panel
  • Documents & e-sign — library, folders, PDF fields, share links, Save to Documents
  • Config — attributes & git layout

Split of responsibilities

  • This CDN — GUI only: embeds, docs, and console assets (agents open the console from their authenticated operator site — see Auth)
  • Your Cloudflare Worker — API only: Durable Objects, WebSockets, intake, email, git sync — product overview

Do not point client sites at someone else’s Worker. Deploy your own, then set data-crm to that URL.
