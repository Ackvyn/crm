Free tier & Worker usage

Ackvyn CRM is aimed at operators who already have a Cloudflare account for their domain and want to run on Workers Free — including the usual 100,000 requests per day per Worker (confirm the current number in Cloudflare’s docs; the quota resets daily). Product story: Why → Built for a free Cloudflare account.

Same idea as Sveltia CMS

With Sveltia CMS, day-to-day editing is mostly browser → GitHub. You typically deploy a small auth Worker so agents can sign in; after that, the CMS script commits content to the repo itself. The Worker is not in the path for every save.

Ackvyn CRM follows that pattern where it can:

  • Console GUI from the CDN (Sveltia-style console.js on your operator site).
  • Auth via your auth Worker (see Auth).
  • Signed-in agents create / update / delete tickets, contacts, and Documents by committing encrypted files to GitHub from the browser — those edits do not need a CRM Worker request for each change.

What must hit the CRM Worker

Some features cannot be pure client-side. They go through your CRM Worker (and Durable Objects where chat needs them):

  • Live chat & presence — WebSockets, visitor sessions, agent online state.
  • Inbound email — Cloudflare Email Routing delivers to the Worker (Email routing). Outbound on free accounts is mailto + CC/BCC the CRM address. Optional Worker send (Settings toggle, off by default) needs Cloudflare Email Sending on a paid Workers plan.
  • Anonymous / form intake — visitors posting without an agent GitHub session.
  • E-sign share sessions — create / complete document signing links (Documents & e-sign).
  • Anything that cannot safely hold your git token in the visitor’s browser — the Worker writes encrypted blobs when the visitor path creates records.

The design goal is to keep that surface as small as possible so Free plan request budgets go to chat and email — not to every ticket field edit in the console.

What stays off the Worker when it can

  • Loading the console, docs, and embeds from the CDN (not your Worker).
  • Agent edits to tickets, contacts, and other git-backed CRM files via GitHub’s Contents API after OAuth.
  • Reading those same encrypted files in the console from GitHub when the agent is signed in.

Mileage may vary

Staying under 100k requests/day is realistic for many small sites, but it is not a guarantee:

  • Other Workers on the same account (CMS auth, redirects, edge logic, etc.) use their own Free allowances — still worth counting total ops load.
  • Busy chat days, many concurrent visitors, or heavy email volume increase CRM Worker invocations.
  • Cloudflare also limits CPU time and other Free-plan features (including Durable Objects). Check Workers limits and Durable Objects pricing/limits for the plan you are on.

If the Free plan is no longer enough, Cloudflare’s Paid Workers plan removes the daily request ceiling. Ackvyn CRM does not require a multi-tenant SaaS upgrade path — you scale the same Worker you already deploy.
