API

Your Worker

The GUI lives on this CDN. The API — chat, tickets, email, git sync — runs on a Cloudflare Worker you deploy to your own account.

Same idea as a CDN-served CMS: operators don’t share a multi-tenant backend. Each shop gets a Worker URL, points embeds at it with data-crm, and keeps crm-data/ in their own git repo. The Deploy button clones worker/ from Ackvyn/crm (that folder’s wrangler.toml), not the monorepo root.

What the Worker does

  • Live chat & presence (Durable Objects + WebSockets)
  • Form / visitor intake when there is no agent GitHub session
  • Inbound email via Cloudflare Email Routing
  • Worker-side encrypted git writes for those paths; public reviews JSON sync

Signed-in agents edit tickets and contacts from the console by committing to GitHub themselves (Sveltia-style) — that path is meant to stay off the Worker. See Free tier & Worker usage.

It does not host the admin console. Agents open the console from your own operator site (iframe or link) after GitHub sign-in — not from a public “Console” button on this marketing site.

Aimed at Cloudflare Free: keep chat and email on the Worker, keep console CRUD on GitHub where possible, and stay mindful of the ~100k requests/day Workers Free allowance. Details in Free tier & usage and Why.

Deploy

Or use Wrangler from the Worker setup guide.

Secrets stay empty in the template on purpose — the Deploy UI must not look like it ships a shared GitHub token or passphrase. You paste your own after deploy.

Secrets after deploy

  • GITHUB_TOKEN — your personal access token with Contents access to the repo in CRM_GIT_REPO. Create it yourself; nothing is pre-filled.
  • CRM_DATA_PASSPHRASE — encrypts crm-data/. Treat it like a high-entropy secret (long random string), not a short memorable password. Use the same value as the Worker secret and in the CRM console. Cloudflare cannot invent a unique one per Deploy click — generate yours below (or openssl rand -base64 32).

Generate a data passphrase

Runs only in your browser (32 random bytes → base64). Copy it into npx wrangler secret put CRM_DATA_PASSPHRASE and the CRM console encryption setting — store it somewhere safe; lost passphrase means you cannot decrypt existing blobs.
